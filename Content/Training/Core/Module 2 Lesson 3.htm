Lesson 3: Building a Sample Form

Lesson Contents In this Lesson, you will learn how to recreate the Form used in the training sample project.

Creating the Form

Now that we've looked at the Form definition and controls, we'll take what we've learned and begin building an actual Form. Let’s begin by logging into Process Director as an admin user. If the Content List doesn’t appear automatically when you log in, then click on the Content List button in the navigation bar of the Admin Profile workspace. Once the Content List appears, create a "Training" folder at the root level of the Default partition. This folder is where we will begin creating the objects for the sample process, and we will start by creating a Purchase Request Form that we will use in building a purchase request approval process.

In the Training folder, create a new folder named "Forms". Once you have created this folder, navigate to it, and, once inside the Forms folder, select “Form Definition” from the Create New… dropdown that is located on the upper right of the Process Director screen. The Create Form screen will open.

The default option for creating a Form is to use the online form designer. You have a number of other options, though we will not discuss them in detail here. The other available options are shown below.

Upload Selected Form: You can create a Form template in Word Format, using the Word Form Builder for Process Director, or by creating an ASCX file in Visual Studio. Using this option will enable you to upload that Form into Process Director.

Use Empty Word Doc Form: This option will create an empty Word Document that you can check out and design using the Word Form Builder.

Use Template Word Doc Form: This option will create a Word Document with some sample fields that you can check out and edit using the Word Form Builder.

Use Empty ASCX Form: This option will create an empty ASCX file that you can check out and design using Visual Studio.

Use Template ASCX Form: This option will create an ASCX file with some sample fields that you can check out and edit using Visual Studio.

Using any of these options are more advanced topics, and are covered in later training courses.

Name the Form by putting the title, "PR Request" in the Form Name box. Now, type in a short description of the form in the Description box. The description that you enter here will appear in the second line of the Content List when you view the contents of the Training folder. Click the OK button to create the new Form definition. The Edit screen for the new Travel Expenses Request Form will appear automatically.

Now we can begin editing our Form.

With the Section in place, move the cursor inside the Section, where the content will go, by clicking inside the Section's content area. Add a table inside the section by clicking the Insert Table button on the toolbar.

The Table Properties dialog box will open.

Set the number of Rows to "2" and the number of Columns to "4", then click the OK button to close the dialog box and create the table inside the Section.

By default, the table will automatically expand to fill the Section as the size of the Section expands or contracts horizontally.

Now, you can perform some simple borders and shading options on the table, if you desire. As described previously in Lesson 2, you can add formatting to the table or the table cells by right-clicking and selecting the properties for the table or cell that you want to configure.

As you can see, adding controls to a Form page is relatively simple and straightforward. Using the same process that we used for the first two controls, add the following controls to column 4 of the table:

Customizing Control Properties

Let's make the VendorZip control shown above shorter than the other Input controls. In the Properties dialog box for Input controls, there is a Columns property, which refers to the number of text characters that can fit horizontally into the control. The default value of the Columns property is "30", i.e., thirty characters can fit horizontally into the control. For the VendorZip control, the set the Columns property to "12", which will decrease the horizontal width of the control.

There is also a Rows property which defines the height of the control in lines, or rows, of text. Using both the Columns and Rows properties, you can define the display width and height of the control.

The Implementer's Guide documentation lists all of the Form controls, and enumerates their properties, with explanations of what each Form control's properties mean. You should refer to the Implementer's Guide for detailed information about all of the Form control properties.

Arrays and Calculations

The next section of the Form will contain an array of items to purchase, as well as some controls that will perform calculations based on the data that users enter into the form. When we are done, this PR Items and Cost section will appear similar to the example below.

First insert the Section control for this section, and set the Name property as "SectionItems" and the Text property as "PR Items and Cost".

Now, inside the section, create a table with three rows and seven columns. Format the table to match the previous two tables we created. Once the table is formatted, add the following column headers to the first row of each column:

  • Row Number
  • SKU
  • Description
  • Quantity
  • UnitPrice
  • Total
  • Row Editing

Now we can begin adding the controls for our array of PR items in the second row of the table:

  • In the first column, insert an Array Start control, and set the Name property to "ArrayItems". Now, insert a Row Number control to the right of the Array Start control.
  • In the second column, insert an Input control. Set the Name property to "SKU" and the Columns property to "10".
  • In the third column, insert an Input control. Set the Name property to "Description", the Columns property to "40", and the Rows property to "3".
  • In the fourth column, insert an Input control. Set the Name property to "Quantity" and the Columns property to "4".
  • In the fifth column, insert an Input control. Set the Name property to "UnitPrice" and the Columns property to "4".
  • In the sixth column, insert an Calculation control. Set the Name property to "ItemPrice" and place the following formula in the Formula property:
    {:Quantity} * {:UnitPrice}
    This formula will calculate the product of the values in the Quantity and UnitPrice fields.
  • In the seventh column, we are going to insert both a nested table and an Array End control. Due to the way the designer works, we are going to insert the Array End control first, because inserting the nested table first will make inserting additional controls in the outer table cell difficult. Insert the Array End control, then, place your cursor to the left of the Array End control. Insert a nested table containing one row and three columns. Format the nested table so that it has no borders or shading.
    • In the first column of the nested table, insert a Move Array Row Up control.
    • In the second column of the nested table, insert a Move Array Row Down control.
    • In the third column of the nested table, insert a Remove Row Inside Array control.
    • None of these controls need any specific configuration.

To complete our array, we need to give users the ability to add new items to the array, so, in the first column of row 3, add an Add Row to Array control. Set the Text property of the control to "Add Items". Since we need to tell Process Director which array to which we need to add rows, set the ArrayName property to "ArrayItems". If you'll remember, this is the Name property of the Start Array control we placed in thesecond row of our array table, and it identifies the array to which rows will added when the Add Row to Array button is clicked.

You have now created a fully functional array that will enable users to add multiple line items for products to be purchased.

In the fifth column of the third row, write the text "PR Total:", and format the cell so that it looks like the cell headers in row one.

In the sixth column of the table, insert a Sum control. Set the Name property to "Subtotal". To identify the column for which we want to calculate the sum, set the Column property to "ItemPrice". The Sum control will calculate a running total of all amounts in the ItemPrice controls in the array, no matter how many rows are created.

The section should now look similar to the example shown above.

Comments and Attachments

The next section of the form will hold a 2 column table, with a Comment Log control in one column, and some attachment controls in the second column.

Leave a blank line below the previous section, and add a new Section control with the Name property of "SectionComments", and a Text property of "Comments/Special Instructions".

Inside the Section control, insert a table with two rows and two columns, and add the column headers "Comments or Special Instructions" and "Supporting Documents" to the first row.

In the first column of the second row, add a Comment Log control with the Name property of "CommentLog" and the Text property of "Add Comment". The Comment Log control will save a list of any comments that users might make as the Form goes through the approval process, as well as the date of the comment and the name of the user who created it.

Since we want to give users the ability to attach invoices, price quotes, or other relevant documents to the Form, insert an Attach Objects control into second row of the second column. Set the Name property as "AttachQuote", the Text property to "Add Document", the Attach Type property to "Process" and the Group Name to "Quotes". Use the [ENTER] key toadd a new line to the table cell, an, below the attachment button, insert a Show Attached Objects control with a Name property of "ShowAttachedQuotes", an Attach Type property of "Process" and a Group Name of "Quotes". The group names used in both attachment controls must be the same, or you will not be able to view the attachments when you add them to the running form.

The Attach Objects control will enable users to find and upload files to Process Director which will be attached to the approval process that will be initiated when the Form is submitted. The Show Attached Objects control will display the attached objects to the users, who can then view or download them.

The section should now be complete, and should look similar to the example below.

Leave a blank line after the previous section, and insert the following controls:

  • An Embedded Section control, with the Name property set to "SectionApproversComments".
  • Type the text, "Approver's Comments (Required if disapproved)", then type [SHIFT] + [ENTER] to enter a line break.
  • A Signature Comments control with a Name property of "SignatureComments".
  • An Embedded Section End control.

Type [ENTER] to add a new paragraph, then insert a Button Area control.

When you're finished, the table should look similar to the example below.

Routing Slip

We've finally come to the last section of the form. Add a blank line at the bottom, then and add the following controls:

  • A Section control with the Name "SectionRouting" and Text of "Routing Slip".
  • Inside the section, add a Routing Slip control.

The Routing Slip gives you a way to visually track where you are in the request’s approval process. Perhaps more importantly, it shows you where you’ve been: who has taken action in the process, what actions they took, and any comments they added along the way. The completed Routing Slip shows us everyone who participated in the process, when they participated, and what the final disposition was. It is automatically created within every process, and can be viewed using the process administration interface, or, as is the case in our sample, directly on the form. The Routing Slip provides a clear, auditable history of the process that can be retrieved and examined for any number of reasons, including audits, process improvement reviews, etc. As an aside, each user can have a signature file associated with their account to display a physical signature in the Routing Slip.
