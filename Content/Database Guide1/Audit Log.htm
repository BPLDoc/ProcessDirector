Audit Log

The audit log contains data recording most events in an installation of Process Director. The audit log displays records in the following CSV format:

Date, Event, User ID, IP, Text, Object Name, Object Type, UID, OID, PID, Impersonate User ID, Impersonate UID

Where the columns are defined as follows:

Date: the date and time the event happened.

Event: the type of event. The event type can be any of the following:

  • Login - A user logged into the system
  • Logoff - A user logged off of the system
  • AuthenticateFailed - A user attempt to authenticate, e.g., logon, failed.
  • DeleteObject - A Content List object was deleted.
  • CreateObject - A Content List object was created.
  • UpdateObject - A Content List object was updated.
  • MoveObject - A Content List object was moved to a different location in the Content List.
  • PasswordChange - A user's password was changed.
  • CreateUser - A new user was created on the system.
  • DeleteUser - An existing user was deleted from the system.
  • CreateGroup - A new user group was created.
  • DeleteGroup - An existing user group was deleted.
  • WebService - A web service was called.
  • TaskCompleted - A process task was completed.
  • TaskReassigned - A process task was reassigned to a different user.
  • TaskRemoved - A process task was removed.
  • Signature - A user provided a signature, usually on a form via a Signature control.
  • RollbackActivity - A Process Timeline Activity was rolled back to a previous activity.
  • CancelActivity - A Process Timeline Activity was canceled.
  • RestartActivity - A Process Timeline Activity was restarted.
  • Restart Timeline - A Process Timeline was restarted.
  • JumpToStep - A Workflow step was jumped to a different step in the Workflow.
  • CancelStep - A Workflow step was canceled.
  • StopStep - A Workflow step was stopped.
  • StopWorkflow - A Workflow was stopped.
  • RestartWorkflow - A Workflow was restarted.
  • UpdateUser - A user's profile information was updated.
  • AddUserToGroup - A user was added to a user group.
  • RemoveUserFromGroup - A user was removed from a user group.
  • RemoveObjectFromParent - A child objectwas removed from the object which contained it. E.g., a document attachment was removed from a process instance.
  • DownloadDocument - A document was downloaded by a user. Usually, this indicates that a user has downloaded a document attachment.
  • ViewFormData - A user viewed an existing form instance's data, usually administratively.
  • PermissionAdded - An object permission was added to a Content List object.
  • PermissionRemoved - An object permission was removed from a Content List object.
  • PermissionUpdated - An object permission was updated on a Content List object.
  • PermissionReplicated - An object permission for a Content List object was replicated to other Content List objects.
  • PermissionGiven - A user permission was granted to a user.
  • PermissionDeniedView - A "Deny" permission to view a Content List object was added to the object.
  • PermissionDeniedModify - A "Deny" permission to modify a Content List object was added to the object.
  • PermissionDeniedDelete - A "Deny" permission to delete a Content List object was added to the object.
  • PermissionDeniedExecute - A "Deny" permission to execute or run a Content List object was added to the object.
  • PermissionDeniedAdmin - A "Deny" permission to perform administrative functions on a Content List object was added to the object.
  • ImpersonateOn - A user began impersonating another user.
  • ImpersonateOff - A user stopped impersonating another user.
  • DelegateOn - Task delegation was turned on for a user.
  • DelegateOff - Task delegation was turned off for a user.
  • UserDisabled - An existing user was disabled.
  • UserEnabled - A disabled user was enabled.
  • AdminReplaceUser - A user was replaced in a task by an administrator.
  • AdminPageAccess - An IT Admin page was accessed by a user.
  • ImportObjects - New Content List objects were imported into the system.
  • ExportObjects - Existing Content List objects were exported from the system.
  • CustomApp - A custom application accessed the system via APIs.
  • FormDataChange - Data was changed in a Form instance.
  • ViewForm - A user viewed an existing Form instance.
  • AdminPageUpdate - A change was made to an IT Admin page's data.
  • UpdateDocumentProps - A user updated the properties of a document or document attachment.
  • RollbackDocument - An edited document or document attachment was rolled back to an earlier version of the document.
  • CancelDocumentCheckout - A user canceled an attempt to check out a document or document attachment.
  • CheckoutDocument - A document or document attachment was checked out, usually for editing.
  • AnonymousAccess - An anonymous user accessed the system.
  • ObjectLock - A Content List object was automatically locked for editing.
  • ObjectUnlock - A Content List object was automatically unlocked from editing.
  • ObjectUnlockOther - A Content List object was manually unlocked.
  • ViewCaseData - A user viewed the data for a Case instance.
  • CaseDataChange - The data stored in a Case instance was changed.
  • CaseModeEntered - A user opened a case instance in case context.
  • ReInit - The Process Director installation was reinitialized, usually from a suspended state after period of inactivity.
  • UserLockedOut - A user was locked out of the system, usually as the result of authentication failures.
  • AddGroupToPartition - A user group was added to a partition to enable access to that partition by members of the group.
  • RemoveGroupFromPartition - A user group was removed from a partition.
  • AddUserToPartition - A user was added to a partition to enable access to that partition.
  • RemoveUserFromPartition - A user was removed from a partition.
  • AddAppEvent - A new application event was added to a Form or Process Timeline definition.
  • UpdateAppEvent - An application event was changed by a user.
  • DeleteAppEvent - An application event was deleted by a user.

User ID: the ID of the user that caused the event.

IP: the IP address the user was logged on with.

Text: an explanation of the event

Object Name: the name of the object involved in the event

Object Type: the type of the object involved in the event

UID:  the UID of the user affected by the event.

OID: the ID of the object affected by the event.

PID: the ID of the process involved in the event.

Impersonate User ID: the ID of the user that was impersonating the user.

Impersonate UID: the UID of the user that was impersonating the user.
