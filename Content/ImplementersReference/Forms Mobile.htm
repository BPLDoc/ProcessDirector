Mobile Forms

Appropriately licensed installations of Process Director 5.XX have the ability to create and use mobile forms. The mobile forms can be viewed in a native BP Logix mobile application that is available from both the Google Play Store for Android devices and the Apple App Store for iOS devices.

Customer needs an appenate account. we create a client for the customer, and get them set up.

Keys are created in appenate account, and have to be placed in vars:

  • nAppenateCompanyID
  • sAppenateIntegrationKey

We set up user account with Mobile app admin user, who can access a bit of the web site. Everyone else has to just use the mobile app. Mobile users can only see stuff from their client. Users can be restricted in what they see.

One datasource per client will be created to access all of their data for forms/screens.

Pull or push: a CT will pull, or appenate can be set to push on a scheduled basis. this will provide lists, etc. to populate form dropdowns.

A CT imports forms from a mobile device as a pull. A Goal will schedule, and the CT will be pulled from a TL prompted by the goal.

Create a form that will be mobile. ON form properties, there's a Download Field list link. The form will be manually loaded into appenate via the spreadsheet (Create Screen), which will config data along with form fields. Forms in appenate are called screens.

Appenate will create a form design from the spreadsheet. You can monkey with it. Except NEVER change the Data Name property of the field on the imported form. Data Name must match the field name in Process Director.
