Classes

Custom programming in Process Director is performed by using a number of classes that are built into the product. Each class in the Process Director library contains all of the relevant properties methods and events. The Process Director SDK classes are listed in the menu bar on the right side of this page.

Common Termination Reasons

The classes associated with Timeline Activities, Process Tasks, and Workflow steps all have a common set of termination reasons associated with every task or activity. The termination reason is set when the task or activity is completed. A list of those classes would include:

  • ProcessTask
  • ProcessTaskUser
  • ProjectActivity
  • ProjectActivityUser
  • Task
  • WorkflowStep
  • WorkflowStepUser

In the Process Director UI, text values are returned for the termination reasons. In Process Director database tables, however, numeric values are stored for the termination reason. Thus, any records returned by database queries or views will display the numeric value for the termination reason. The numeric and text values for each termination reason, along with a description of each value, is provided in the table below.

NUMERIC VALUE UI TEXT VALUE DESCRIPTION
0 Not Set Not Set
1 Completed Normal completion
2 Timeout Timeout occurred on this step or activity
3 Not Required The user did not finish because conditions allowed the task to complete
4 Admin Cancelled The step, activity or user was cancelled
5 Reassigned The user task was reassigned to another user
6 Not Required The activity was not needed because of the “Needed When” conditions
7 Notified The user was notified by email in an Notify task
8 Stopped The Timeline was stopped at this activity
9 Canceled An iteration break condition was met on this running activity causing it to be stopped
10 Parent Restarted An iteration restart condition was met on this running activity causing it to be stopped
