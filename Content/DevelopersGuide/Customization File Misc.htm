Miscellaneous Variables

There are a variety of miscellaneous custom variables that you can set to determine how some Process Director features or functions work.

AllowRichTextTemplate

When this flag is set to true, Process Director will attempt to transfer Rich Text content into PDF form fields that are set to accept Rich Text content. There are limitations to this ability, which are explained in the PDF Rich Text Field Support section of the PDF Custom Tasks topic.

Example

public override void SetSystemVars(BPLogix.WorkflowDirector.SDK.bp bp)
{
    bp.Vars.AllowRichTextTemplate = true;
}

 AlwaysFindTaskForForms

 When this flag is set to true, forms will attempt to associate with a task when opened. If this is left as the default (false), forms will only associate with a task when opened from a task list knowledge view.

Example

public override void SetSystemVars(BPLogix.WorkflowDirector.SDK.bp bp)
{
    bp.Vars.AlwaysFindTaskForForms = true;
}

        "<link href=\"http://sample.bplogix.com/custom/sample-bp-icon.jpg\"

        rel=\"apple-touch-icon\"><link rel=\"stylesheet\" type=\"text/css\"

        href=\"theme.css\">";
}

 fAuthWindows

This variable, when set to true, enables the use of Windows login security. The default value for this variable is "false", and is set in the XSD file.

You usually do not turn on this feature on by changing the value of this System Variable. Instead, this feature is enabled in the product in the User Authentication settings by setting the Enable Windows Authentication dropdown to "True" .Windows login security cannot be implemented until the Enable Windows Authentication setting is set to "True".

Example

public override void SetSystemVars(BPLogix.WorkflowDirector.SDK.bp bp)
{
    // Windows authentication is disabled
    bp.Vars.fAuthWindows = false;
}

While this variable is set to "true" by default, Windows Integrated Security is set to "false' by default. You usually turn on this feature in the product in the User Authentication settings by setting the Enable Windows Authentication dropdown to "True". Even though the fAuthWindowsIntegrated variable is set to true by default, windows login security cannot be implemented until the Enable Windows Authentication setting is set to "true".

While this may substantially speed up the search operation, it also reduces the accuracy of the search results, as search terms that are not included in the first 256 characters of the field will be ignored.

 FormEditorConfig

This variable specifies the location of a custom JavaScript configuration file to use, in addition to the default file, and primarily enables you to customize the fonts used by the online form designer. This variable will universally effect the available controls for every form definition that uses the Online Form Designer.

Example

In this example, we will use the FormEditorConfig variable to make a font named "Lobster" available for use in the Online Form Designer, and to display for end users in the completed form at run time.

public override void PreSetSystemVars(BPLogix.WorkflowDirector.SDK.bp bp)
{
    // Set a custom configuration file for the Form Editor
    bp.Vars.FormEditorConfig = "/custom/cust_config.js";
}

Notice that you need to use full client path, without using a "~", in the URL to the Configuration file.

For this to work, you must create cust_config.js as:

CKEDITOR.editorConfig = function (config) {
    config.contentsCss = "/custom/cust_fonts.css";
    config.font_names = 'Lobster;' + config.font_names;
}

Then create cust_font.css and, using the sUseCSS custom variable, add a reference to the CSS file to the Custom Variables file as:

bp.Vars.sUseCSS.Add("~/custom/cust_fonts.css");

Note that the use of the tilde (~) is acceptable for the sUseCSS variable.

The cust_fonts.css file can use a @font directive to specify the new custom font you'd like to make available to designers:

@font-face {
    font-family: "Lobster";
    font-style: normal;
    src: local("Lobster-Regular"), url("/custom/Lobster-Regular.ttf") format("truetype");
}

This CSS configuration assumes that you have added the TTF file for the Lobster font to the "custom" folder of the installation.

Once you have completed these steps, the Lobster font will be available for use in the Online Form designer, and will display properly in Process Director for end users.

By default, file uploads for Form attachments occur synchronously with Form submissions, meaning that the Form submission cannot complete until all files are uploaded.
